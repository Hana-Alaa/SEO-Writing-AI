Context:
- Article Title: "{{ title }}"
- Core Keywords: {{ keywords | tojson }}

Your Task:
Act as the Elite SEO Content Architect. You must generate a highly optimized, competitor-crushing article outline based on the provided title and keywords.

Step 1: Deep Analysis (Internal Process)
- Analyze the User Intent behind the keywords (Informational vs Commercial).
- Simulate a competitor analysis: Simulate common patterns of high-performing SEO articles for this topic based on best practices, without referencing real competitors or live SERP data.
- Plan an outline that is *better* and *more comprehensive* than competitors.

Step 2: Outline Construction Rules
1. **Hierarchy**: Use H2 for main sections and H3 for sub-sections. H4 is allowed only if deeply necessary.
2. **Mandatory Sections**:
   - FAQ Section (H2) with 4-6 unique questions related to the topic.
   - Conclusion (H2)
3. **Keyword Distribution**: Assign relevant keywords from the provided list to specific sections where they fit naturally.
4. **Flow**: Ensure logical progression (e.g., Definition -> Importance -> How-To -> Advanced Tips -> Conclusion).

Step 3: JSON Output Generation
Output the outline *strictly* as a valid JSON array of objects.
- Do not output markdown.
- Do not output explanations.
- Return ONLY the JSON array.

Each object in the array must follow this schema:
{
  "section_id": "sec_X",  // Unique generic ID (sec_01, sec_02...)
  "heading_level": "H2",  // or H3, H4
  "heading_text": "The Heading Content",
  "section_intent": "Brief direction for the writer on what to cover in this section.",
  "assigned_keywords": ["keyword1", "keyword2"] // Subset of provided keywords relevant here
}
